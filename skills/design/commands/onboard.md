Design and improve onboarding flows.

## Steps

1. Read `conventions/design/design-context.md` for target audience
2. Evaluate current first-run experience:
   - What does a new user see first?
   - How quickly do they reach value?
   - Is setup required? Can it be deferred?
3. Design onboarding improvements:
   - Empty states: show clear CTAs, not just "No data yet"
   - Progressive disclosure: introduce features contextually, not all at once
   - Inline guidance: tooltips on first interaction, not a tutorial wall
   - Quick wins: let the user accomplish something in < 60 seconds
4. Check edge cases:
   - Returning user who forgot the UI
   - User who skipped onboarding
   - User on mobile vs desktop
5. Reference `references/design/ux-writing.md` for copy guidance
6. Reference `references/design/interaction.md` for interaction patterns

## Rule
The best onboarding is no onboarding — make the UI self-explanatory.
