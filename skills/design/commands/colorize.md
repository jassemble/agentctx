Add strategic color to monochromatic or visually flat designs.

## Prerequisites

1. **Read reference material**:
   - Read `.agentctx/context/references/color.md`

2. Identify what to colorize:
   - If argument provided, focus on that area: $ARGUMENTS
   - Otherwise, scan for pages that feel gray, flat, or lacking visual warmth

## Steps

### Identify Where Color Adds Value

- **Semantic meaning**: success (green), error (red), warning (amber), info (blue)
- **Hierarchy**: draw attention to primary actions, important content
- **Categorization**: distinguish sections, types, or states visually
- **Emotional tone**: warmth, energy, trust, personality

### Apply the 60-30-10 Rule

- **Dominant color (60%)**: primary brand or most-used accent
- **Secondary color (30%)**: supporting color for variety
- **Accent color (10%)**: high contrast for key moments
- Stay within 2-4 colors beyond neutrals — more is not better

### Implementation

- Color primary CTAs and links; colorize key icons for recognition
- Replace pure grays with tinted neutrals (warm or cool) for sophistication
- Use subtle background tints to separate sections
- Add colored left/top borders on cards for accent without overwhelming
- Use OKLCH for perceptually uniform color scales

### Check Contrast

- Text on colored backgrounds must meet WCAG AA (4.5:1 normal text, 3:1 large text)
- Do not use gray text on colored backgrounds — use a darker shade of the background color instead
- Do not rely on color alone for meaning — pair with icons, labels, or patterns

## Important

- More color does not mean better — every color should have a purpose
- Strategic color beats rainbow chaos every time
- Do not default to purple-blue gradients (AI slop aesthetic)
