Simplify by removing unnecessary complexity.

## Steps

1. Read `conventions/design/principles.md` — Quick Rules section
2. For the target area ($ARGUMENTS), audit every visible element:
   - Does this element earn its place? What happens if removed?
   - Can two elements be combined into one?
   - Is information duplicated across the page?
3. Apply progressive disclosure:
   - Show essentials upfront, reveal details on demand
   - Collapse secondary actions into menus
   - Use expandable sections for advanced options
4. Reduce visual noise:
   - Remove decorative elements that don't communicate
   - Simplify borders, shadows, gradients — use fewer visual layers
   - Reduce color variety — fewer colors, more consistency
5. Streamline user flows:
   - Can any step be removed from this workflow?
   - Can defaults eliminate decisions?
   - Can we auto-detect instead of asking?

## Rule
If you can't articulate why an element exists, remove it.
